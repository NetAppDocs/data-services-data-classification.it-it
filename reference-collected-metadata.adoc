= Metadati raccolti da fonti di dati in NetApp Data Classification
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
NetApp Data Classification raccoglie metadati durante la scansione delle tue origini dati e dei tuoi sistemi. La maggior parte dei metadati è disponibile, ma alcuni attributi non sono disponibili per tutti i protocolli.

[cols="15,25,25,25"]
|===
|  | *Metadati* | *CIFS* | *Non è vero* 


.3+| *Timbri temporali* | _Ora di creazione_ | Disponibile | Non disponibile (non supportato in Linux) 


| _Ultima ora di accesso_ | Disponibile | Disponibile 


| _Ultima modifica_ | Disponibile | Disponibile 


.2+| *Autorizzazioni* | _Apri permessi_ | Se il gruppo "TUTTI" ha accesso al file, questo è considerato "Aperto all'organizzazione" | Se "Altri" ha accesso al file, questo è considerato "Aperto all'organizzazione" 


| _Accesso utenti/gruppi_ | Le informazioni sugli utenti e sui gruppi vengono prese da LDAP | Non disponibile (gli utenti NFS sono solitamente gestiti localmente sul server, pertanto lo stesso individuo può avere un UID diverso in ogni server) 
|===
[NOTE]
====
* La classificazione dei dati non estrae l'"orario dell'ultimo accesso" dalle fonti dati del database.
* Nelle versioni precedenti del sistema operativo Windows (ad esempio, Windows 7 e Windows 8), la raccolta dell'attributo "ora dell'ultimo accesso" è disabilitata per impostazione predefinita perché può influire sulle prestazioni del sistema. Quando questo attributo non viene raccolto, le analytics di Data Classification basate sull'"ora dell'ultimo accesso" ne risentiranno. Puoi abilitare la raccolta dell'ora dell'ultimo accesso su questi sistemi Windows meno recenti se necessario.


====


== Ultimo accesso

Quando Data Classification estrae dati dalle condivisioni di file, il sistema operativo lo considera un evento di accesso e aggiorna di conseguenza l'ora dell'ultimo accesso. Dopo la scansione, Data Classification tenta di ripristinare l'ora dell'ultimo accesso al timestamp originale. Se Data Classification non ha i permessi di scrittura degli attributi in CIFS o i permessi di scrittura in NFS, il sistema non può ripristinare l'ora dell'ultimo accesso al timestamp originale.

Per impostazione predefinita, Data Classification non esegue la scansione dei file se non dispone delle autorizzazioni di scrittura per ripristinare il timestamp originale. Se i tuoi dati possono tollerare che l'ora dell'ultimo accesso non venga ripristinata al timestamp originale, abilita l'opzione *Scansione senza autorizzazioni di scrittura* nella pagina di configurazione del sistema.

image:screenshot_scan_missing_permissions.png["Uno screenshot che mostra l'interruttore che puoi usare per fare in modo che Data Classification analizzi i tuoi file anche se non ha le autorizzazioni corrette per ripristinare l'ora dell'ultimo accesso al timestamp originale."]

Questa funzionalità è applicabile ai sistemi ONTAP locali, Cloud Volumes ONTAP, Azure NetApp Files, Amazon FSx for NetApp ONTAP e condivisioni file di terze parti.


NOTE: I volumi ONTAP configurati con SnapLock hanno autorizzazioni di sola lettura e inoltre non possono ripristinare l'ora dell'ultimo accesso al timestamp originale.
